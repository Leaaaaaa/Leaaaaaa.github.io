web自动化测试练习

发表于 2019-11-24

什么样的项目适合Web自动化测试

  1. 需求变动不频繁
  2. 项目周期长
  3. 项目需要回归测试
  • 基于元素属性

find_element_by_id()
find_element_by_name()
find_element_by_class_name()

  • 基于元素的标签名称

find_element_by_tag_name()

  • 定位超链接标签

find_element_by_link_text()
find_element_by_partial_link_text()

  • 定位一组元素返回的是列表,从0开始

访问必须指定下标，下标从零开始
例如：find_elements_by_tag_name(“input”)[1].send_keys(“1111”)

xpath定位 ：driver.find_element_by_xpath()

  1. 绝对路径：从最外层元素到指定元素之间所有经过元素层级的路径：绝对路径以/ 开始，中间不能跳跃元素, 使用firebug可以快速生成元素xpath的绝对路径

  2. 相对路径：从第一个符合条件元素开始（一般配合属性区分）：相对路径以 // 开始，//后面加上标签名称或*（所有标签）,使用friebug扩展插件firepath 可以快速生成元素相对路径

  3. 利用元素属性性

    说明：快速定位元素，利用元素唯一属性

    示例：//*[@id=‘kw’]

  4. 层级与属性结合
    说明：要找的元素没有属性，但是他的父级有
    示例：//*[@id=‘p1’]/input

  5. 属性与逻辑结合
    说明：解决元素之间个相同属性重名问题
    示例：//*[@id=‘telA’ and @class=‘telA’]

  6. xpath扩展

  1. //*[text()=“xxx”] 文本内容是xxx的元素

  2. //*[starts-with(@attribute,’xxx’)] 属性以xxx开头的元素

  3. //*[contains(@attribute,’xxx’)] 属性中含有xxx的元素

css定位方法：
driver.find_element_by_css_selector()

  1. Id 选择器：
    说明：根据元素id属性来选择,必须以#修饰
    格式：#id 如： #userA <选择id属性值为userA的所有元素>
  2. class 选择器：
    说明：根据元素class属性选择
    格式：.class 如：.telA<选择class属性值为telA的所有元素>
  3. 元素选择器：
    说明：根据元素的标签名选择
    格式：element（元素标签名），如：input <选择所有input元素>
  4. 属性选择器：
    说明：根据元素属性名和值来选择
    格式：[attribute=value] 如：[type=“password”] <选择所有type属性为password的值>
  5. 层级选择器：
    说明：根据元素的父子关系来选择
    格式：element>element 如：p>input <返回所有p元素下所有的input元素>
    提示：> 可以代替空格 如： p input 或者
    p [type=‘password’]
  6. Css 延伸
  1. input[type^=‘p’] 说明：type属性以p字母开头的元素
  2. input[type$=‘d’] 说明：type属性以d字母结束的元素
  3. input[type*=‘w’] 说明：type属性包含w字母的元素
