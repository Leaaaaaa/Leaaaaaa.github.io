TouchAction

发表于 2019-11-18
  • 高级手势

TouchAction
轻敲、按下、抬起、等待、长按、手指移动

导包：
from appium.webdriver.common.touch_action import TouchAction

  • 使用步骤：
  1. 创建TouchAction对象
  2. 通过对象调用想要执行的手势
  3. 通过perform()执行
  • 手指轻敲操作

应用场景：模拟手指对某个元素或坐标按下并快速抬起，比如点击（100，100）的位置
参数：
element ： 元素
x ： x坐标
Y ： y坐标
TouchAction(driver).tap(element= None, x = None, y = None).perform()
可选元素轻敲也可选坐标轻敲

  • 按下和抬起

应用场景：模拟手指一直按下，模拟手指抬起。可以用来组合成轻敲或长按的操作

参数：

El：元素

x：x坐标

y：y坐标

按下：
TouchAction(driver).press(element= None, x = None, y = None).perform()

抬起：
TouchAction(driver).release().perform()

其他形式：
TouchAction(driver).press(element= None, x = None, y = None).release().perform()

  • 等待

应用场景：模拟手指等待，比如按下后等地啊5秒之后再抬起

参数：

Ms：暂停的秒数

TouchAction(driver).wait(ms=0).perform()

按下后后等待两秒抬起
TouchAction(driver).press(element= None, x = None, y =None).wait(2000).release().perform()

  • 长按

应用场景：模拟手指对元素或坐标的长按操作，比如，长按某个按钮弹出菜单

参数：

El：元素

x：x坐标

y：y坐标

Duration：长按时间，单位，毫秒

TouchAction(driver).long_press(element= None,x=None,y=None,duration=10000).perform()

长按两秒后抬起
TouchAction(driver).long_press(element= None,x=None,y=None,duration=20000).perform()

额外补充：长按==按下.等待.抬起
//用等待和长按可以一样的效果，不需要松手

  • 移动

模拟场景：模拟手指移动操作，比如，手势解锁需要先按下，再移动

参数：

el：元素

x：x坐标

y：y坐标

TouchAction(driver).move_to(el = none, x = none, y = none).perform()

滑动和拖拽事件

发表于 2019-11-18
  • swipe滑动事件

    概念：
    从一个坐标位置滑动到另一个坐标位置，只能是两个点之间的滑动。

    参数：
    start_x ： 起点x轴的坐标

    start_y ： 起点y轴的坐标

    end_x ： 终点x的坐标

    end_y ：终点y的坐标

    duration :滑动这的操作一共持续的时间长度，单位 ms

    driver.swipe(start_x, start_y, end_x, end_y，duration = None )

    小结：
    距离相同时，持续时间越长，惯性越小
    持续时间相同时，手指滑动的距离越大，实际滑动的距离也越大

  • scroll滑动事件：

    从一个元素滑动到另一个元素，直到页面自动停止

    参数：

    origin_el : 滑动开始的元素

    destination：滑动结束的元素

    driver.scroll(origin_el , destination_el )

    小结：
    不能设置持续时间，惯性很大

  • drag_and_drop 拖拽事件

    从一个元素滑动到另外一个元素，第二个元素替代第一个原色原来的位置

    参数：

    origin_el : 滑动开始的元素

    destination：滑动结束的元素

    driver.scroll(origin_el , destination_el )

    小结：
    不能设置持续时间，没有惯性

  • 滑动和拖拽事件的选择：

    有惯性，传入元素
    scroll无惯性，传入元素
    drag_and_drop
    有惯性，传入坐标
    Swipe ，设置较短的duration时间
    无惯性，出入坐标
    Swipe，设置较长的duration时间

元素等待

发表于 2019-11-17 更新于 2019-11-20
  • 元素等待：

由于一些原因我们要定位的元素没有立刻出来，此时定位元素会报错，所以设置一个时间，等待元素师，等待元素出来之后在进行定位
1.由于网络原因
2.服务器处理原因
3.电脑配置原因
元素等待类型：显示等待，隐式等待

  • 隐式等待：

    等待元素加载指定的市场，超出市场抛出NoSuchElementException异常

    步骤：在获取driver对象后，调用implicitly_wait（）方法即可

    例如：driver.implicitly_wait(5)

    driver.find_element_by_…().click()

    作用:在设置了超时时间之后，后续所有的定位元素的方法都会在这个时间内等待元素出现，如果出现了，直接进行后续操作，如果没有出现，报错，NoSuchElementException

  • 显示等待：

    针对所有定位的超时时间设置为不同的值的时候，在一段时间内不同的时间间隔内多次尝试定位元素
    步骤：

  1. 导包 from selenium.webdriver.support.wait import WebDriverWait

  2. 创建webdriverwait对象

  3. 调用webdriverwait对象的until方法

    例如：wait = WebDriverWait(driver, 15).until(lambda x: x.find_element_by_xpath(“//*[@text=’发现’]”))
    wait.click() 时间单位：秒

    作用：在设置了显示等待之后，可以等待一个超时时间，在这个超时时间内进行查找，默认每0.5秒照一次，频率也可以自己设置，一旦找到这个元素，直接进行后续操作，如果没有找到，报错。TimeOutException

  • 两个等待的选择：
    作用域：显示等待为单个元素有效，隐式等待为全局元素
    方法：显示等待方法封装在webdriverwait中，饮食等待直接通过driver实例化对象调用
    关于sleep形式：sleep是一个固定死一个时间
    而元素等待是可以让元素出来的第一时间进行操作，sleep可能造成不必要的浪费

# 导入appium库

from appium import webdriver

# 创建字典

desired_caps= dict()

# 设备类型

desired_caps[‘platformName’] = ‘Android’

# 设备版本

desired_caps[‘platformVersion’] = ‘6.0.1’

# 安卓随意写，苹果按手机型号写，如iPhone6

desired_caps[‘deviceName’] = ‘test’

# app的包名

desired_caps[‘appPackage’] = ‘com.android.settings’

# app的启动名（界面名

desired_caps[‘appActivity’] = ‘.SubSettings’

# 创建一个driver对象 我们写的Python代码会访问本机的appium服务器，并获取driver对象。appium会将我们的driver对象调用方法转化为post请求，提交给appium服务器。appium通过接受到的post请求发送给手机（虚拟器），再由手机执行

driver = webdriver.Remote(‘http://localhost:4723/wd/hub', desired_caps）

  • 获取APP启动时间

adb shell am start -W +包名/启动名（打开一个应用程序并返回时间）

Thistime：该界面（activity)启动的耗时时间

totaltime：应用自身启动耗时=thistime+应用application等待资源启动时间

Waittime：系统启动应用耗时=totaltime+系统资源启动时间

  • 如何使用appium打开任意app

1.打开要测试的应用

2.使用adb命令获取包名和启动名

3.修改desired_caps字典中的appPackage和appActivity的参数

  • 在一个应用内跳转到另一个应用

driver.start_activity(“包名”, “启动名”)

  • 在跳转时，想要输出其包名或启动名,或者想要在报告中展示对应的信息

driver.current_package

driver.current_activity
