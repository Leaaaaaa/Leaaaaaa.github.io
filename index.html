appium

发表于 2019-11-16

##有关appium的学习笔记
###基本信息配置
# 导入appium库

from appium import webdriver

# 创建字典

desired_caps= dict()

# 设备类型

desired_caps[‘platformName’] = ‘Android’

# 设备版本

desired_caps[‘platformVersion’] = ‘6.0.1’

# 安卓随意写，苹果按手机型号写，如iPhone6

desired_caps[‘deviceName’] = ‘test’

# app的包名

desired_caps[‘appPackage’] = ‘com.android.settings’

# app的启动名（界面名

desired_caps[‘appActivity’] = ‘.SubSettings’

# 创建一个driver对象 我们写的Python代码会访问本机的appium服务器，并获取driver对象。appium会将我们的driver对象调用方法转化为post请求，提交给appium服务器。appium通过接受到的post请求发送给手机（虚拟器），再由手机执行

driver = webdriver.Remote(‘http://localhost:4723/wd/hub', desired_caps）

  • 获取APP启动时间

adb shell am start -W +包名/启动名（打开一个应用程序并返回时间）

Thistime：该界面（activity)启动的耗时时间

totaltime：应用自身启动耗时=thistime+应用application等待资源启动时间

Waittime：系统启动应用耗时=totaltime+系统资源启动时间

  • 如何使用appium打开任意app

1.打开要测试的应用

2.使用adb命令获取包名和启动名

3.修改desired_caps字典中的appPackage和appActivity的参数

  • 在一个应用内跳转到另一个应用

driver.start_activity(“包名”, “启动名”)

  • 在跳转时，想要输出其包名或启动名,或者想要在报告中展示对应的信息

driver.current_package

driver.current_activity

adb命令

发表于 2019-11-16

##有关adb的相关命令：
1、启动adb服务

adb start-server

2、终止adb服务

adb kill-server

3、进入adb运行环境

adb shell

4、获取帮助 里面有adb的各种命令和参数的介绍

adb help

5、查看adb版本

adb version
6、以root权限重启adb

adb root

7、将system分区重新挂在为可读写分区，此命令在操作系统目录时很重要

adb remount

8、重启设备,可选参数进入bootloader(刷机模式)或recovery(恢复模式)

adb reboot [bootloader|recovery]

apk相关:

1、安装apk

adb install test.apk -r 覆盖安装，保留数据和缓存文件 -d 解决低版本version问题 -s 安装apk到sd卡

2、卸载apk

adb uninstall -k

可选参数-k的作用为卸载软件但是保留配置和缓存文件

3、查看app相关所有信息，包括action,codepath,version,需要的权限等等信息

adb shell dumpsys package

4、查看app的路径

adb shell pm path

package:/data/app/com.tencent.test-1/base.apk

5、查看apk的版本信息

adb shell dumpsys package | grepversion

6、查看某个app进程的相关信息

adb shell ps <package_name|PID>

7、杀掉某个进程，一般用于模拟某个bug复现

adb shell kill pid Number

8、查看某一个app的内存占用

adb shell dumpsys meminfo <package_name|PID>

title: Mac使用Charles有关问题
date: 2019-11-01
tags: - Charles
- Chrmoe
–
在前几天，我开始尝试学习各种测试以及抓包工具。在使用Charles时，遇到一些问题，于是想要记录下来。
首先是关于证书问题。当我在Mac上安装好了Charles并且相关代理配置好了以后，我发现我的Safari因为安全问题不能连接服务器而打不开网页了。然后发现是证书问题，于是我按照教程去下载安装安装证书。但是问题出现了，我点击install Charles root certificate的时候，弹出了钥匙串，但是里面没有Charles的证书，也就无法进行信任授权。多次尝试和搜索也不知道为什么里面没有那个证书。最后成功的办法是点击save Charles root certificate，保存到电脑中，再把那个证书拖进钥匙串里，就可以信任授权啦。
后来我又遇到一个和Charles有关的问题。我在Chrome里面下载了一个switchyOmega插件，然后在Charles运行时选择Charles代理，进行Chrome上的抓包工作。但是后来我发现我的Chrome无法连接服务器了.百度上说是代理的问题，然后发现关闭了Charles之后，应该把Chrome里面的代理换回原来的系统代理，然后Chrome就可以正常运行了。
